# deploy 스킬

수강생 결과물을 인터넷에 공개한다. 빌드·zip·업로드는 `deploy.sh` 가 전부 처리하고,
**마지막 줄에 배포 주소**를 출력한다. 수강생 환경에는 배포 토큰·권한이 없다(엔드포인트가 처리).

업로드는 **최대 몇 분** 걸릴 수 있다. 스크립트가 끝날 때까지 **기다린다**.

## 절차 — 묻지 말고, 현재 폴더에서, 한 번에

1. 시작 안내(수강생에게): "지금 인터넷에 올리는 중이에요. 최대 몇 분 걸릴 수 있어요. 잠깐만 기다려 주세요!"
2. 현재 작업 폴더(프로젝트 루트)에서 실행한다 — cd 로 이동하지 않는다:
   ```bash
   bash .agent/skills/deploy/deploy.sh
   ```
   - 몇 분 걸려도 **중간에 끊지 말고 끝까지 기다린다.**
3. 스크립트가 끝나면 **마지막 줄(주소)**을 수강생에게 전달:
   > "다 됐어요! 이 주소를 누르면 누구나 볼 수 있어요 🎉 <주소>
   >  내용을 바꾼 뒤 '배포해줘'라고 하면 같은 주소에 다시 올려드려요."

## 오류 처리 (다음 행동만)

- 스크립트가 0이 아닌 코드로 끝나거나 주소가 안 나오면: 빌드 로그(stderr)를 보고 코드 문제면 직접 고쳐 다시 실행.
  수강생에겐 "잠깐 고치고 다시 올리는 중이에요"만.
- 업로드/서버 문제로 보이면(타임아웃·연결 실패): 오류메시지를 보여주면서 "강사에게 문의하세요" 라고 안내.
